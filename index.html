What to look at

  • Placement: the cross-section module is a sibling below DetailExpand in the sticky right column — not inside the hero.
  • Self-dimension exclusion: focused item is a Cocktail, so tabs are Account · Category · Price · Brands · Flavor — no Cocktails tab. Account is always first.
  • Scoping: every tab reads from Margarita’s account set (existing focused-item → scoped-account-set pattern), not the whole market.
  • Signal-aware header: Margarita over-indexes (the ◆ badge), so the cross-section reads “over-indexing vs LA Bars” — not “trending”. Trending items get the real-date “May 2026 vs May 2025” copy instead (see the Social screen).
  • Selecting a tab keeps the focused row highlighted on the left (indigo left-border).
  • Empty states are honest — see the Brands tab: names the focused scope, never falls back to unfiltered data.
  • Account chain-type badges spell out Independent / Regional / National / Chain (reused from VenueTable’s Chain Type column).
  • On Menus Now stays — it’s part of the unchanged DetailExpand above; the cross-section module is purely additive below it.

What to look at

  • Sheet order: focused hero first, then the cross-section subsection below it — matching the split right column.
  • All tabs fit in one row (equal-width segments) — no horizontal scroll, so no options are hidden off-screen. Wraps to a second row only if labels ever grow.
  • Close lives on the grab handle (tap the handle / down-chevron, plus scrim & Escape) — clears the focused item and ?itemId, per current detail behavior. No separate corner button.
  • Same module, same tab language as iPad — one component, two layouts.

Signal-aware header copy — decision & flagged bug

  • Header mirrors the focused item’s signal: an over-index item → {Dimension} over-indexing vs {benchmark} (e.g. “Cocktails over-indexing vs LA Bars”); a trending item → {Dimension} trending: {Month YYYY} vs {Month YYYY}. Never label an over-index item “trending”.
  • Trends real-date fix: for trending items, lift the precise legend dates into the header (May 2026 vs May 2025), replacing today’s vague “vs Year Ago”.
  • ⚠ Bug to fix in implementation: the Sales r3 (trailing-3-month) window currently maps to “vs Year Ago”. It should read the real comparison, e.g. vs Feb 2026. Shown corrected here; confirm phrasing with product.
